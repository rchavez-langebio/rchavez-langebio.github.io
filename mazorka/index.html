What is mazorka?

mazorka is the high performance computing (HPC) cluster at Langebio.

mazorka is:

  • 1 Master node
    • 2 Intel(R) Xeon(R) E5-2650v3 processors (10 cores and 20 threads per processor = 40 threads per node)
    • 128 GB (8 x 16GB) DDR4-2133 2Rx4 LP ECC REG RoHs
    • 5 SSDs 100GB SATA 3.0 RAID 5
    • 2-port Gigabit standard LP NIC card
    • Infiniband FDR-10
  • 16 Nodes
    • 2 Intel(R) Xeon(R) E5-2650v3 processors (10 cores and 20 threads per processor = 40 threads per node)
    • 64 GB (8 x 8GB) DDR4-2133 1Rx4 LP ECC REG RoHs
    • 1 Hard drive 1.0TB SATA 6 GB/s 7.2K RPM 128M 3.5”
    • Infiniband FDR-10
  • 5 high memory nodes
    • 2 Intel(R) Xeon(R) E5-2650v3 processors (10 cores and 20 threads per processor = 40 threads per node)
    • 768 GB (24 x 32GB) DDR4-2133 4Rx4 LP ECC LRDIMM (38.4 GB per thread)
    • 1 Hard drive 4TB SATA 6Gb/s 7.2K RPM 128M 3.5”
    • Infiniband FDR-10
  • 2 Nodes
    • 4 Intel(R) Xeon(R) E7-4820v2 processors (8 cores and 16 threads per processor = 64 threads per node)
    • 252 GB RAM
    • 1 Hard drive 1 TB
    • Infiniband FDR-10
  • 21 nodes
    • 2 Intel(R) Xeon(R) E5530 processors (4 cores and 8 threads per processor = 16 threads per node)
    • 20 GB to 64 GB RAM
    • Ethernet

Submitting jobs to mazorka

mazorka uses the Portable Batch System (PBS) job scheduler. More info on PBS options can be found here and here .

Queues

mazorka has two queues, default and ensam . A queue is requested with #PBS -q default or #PBS -q ensam .

Resources

Resources (nodes, processors, memory and time) are requested with the header #PBS -l :
- Nodes and processors per node are requested with: #PBS -l nodes=num_nodos:ppn=num_proc_por_nodo
- Memory is requested with: #PBS -l mem=memory:vmem=memory
- Time is requested with #PBS -l walltime=HH:MM:SS

Job name

A name for the job can be declared with -N job_name .

Environment variables

You should export all environment variables to your job using #PBS -V .

Redirecting standard output and standard error

By default mazorka creates a file for standard output named job_name.o job_id and a file for stadard error named job_name.e job_id in $O_PBS_WORKDIR. A different path name can be specified with #PBS -o /path/file_name and #PBS -e /path/file_name .
